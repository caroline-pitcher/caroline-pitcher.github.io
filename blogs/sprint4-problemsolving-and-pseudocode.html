Problem solving and Pseudocode

A time I was blocked on a simple problem

When I was doing the Super FizzBuzz kata, one thing that blocked me for ages was I was trying to use division to solve it instead of remaindering. We had to get the code to give us a particular message if we put in a mulitple of 3 (“Fizz), or a mulitple of 5 (“Buzz”), or if we put in a multiple of 15 (“FizzBuzz”). So I spent a lot of time trying to solve this using division which seemed logical for me. One issue is that I had to include extra code for the number 1 as that can be divided by 3, 5 or 15 but in the code it wasn’t supposed to, it was supposed to come back as 1. I used lots of googling, trying things and reading error messages. In the end as you may have already worked out, this approach using division wasn’t a good idea. I could get the “Fizz” and “Buzz” messages to work but not the “FizzBuzz” because 15 can be divided by both 3 and 5 so I got “Fizz” or “Buzz” but not “FizzBuzz”. In the end I looked at the sample solution and realised that the remaindering method was a lot easier and was going to result in shorter code as well. I did still learn a lot from trying the other option but it was important to remind myself that there isn’t just one way to do things and if something doesn’t work it’s often better to move on rather than becoming fixated on one option.

A time I solved a problem in an elegant way

I’m going to use a particular example that I still think is a good idea even though it ultimately didn’t past the test and I had to give it up. This was a function we had to write called deBee where we had a sentence that said something like “buzz help buzz I’m being buzz attacked by bees buzz” and we needed to remove all the buzzes and return the sentence without them, eg. “help I’m being attacked by bees”. The kata gave the instruction to use the techniques of split and filter, so split the sentence up into individual words, filter out the buzzes and then join the rest of the words back up into a new sentence. However while I was googling and reading online I saw that you could use the replace technique and make it replace a word with nothing which seemed like it would do the same thing but with less steps. I continued to try this idea out in lots of ways and used google and console logging to get the result which seemed to work fine. Unfortunately I couldn’t get it to pass the test but I still think it could have worked. I did learn a lot more techniques to do with strings in this challenge and I think when I have a better understanding of JavaScript it will be possible to try ideas like this one out with more knowledge.
